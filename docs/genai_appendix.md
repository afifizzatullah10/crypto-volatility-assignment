## What Was Reviewed and Verified Manually

All AI-generated code was verified by:

1. **Logical review** — reading the generated code to confirm it matches the plan's specification
2. **No look-ahead bias check** — confirmed that `features/core.py` uses only data ≤ T, and `scripts/replay.py` generates forward labels only after all ticks are processed
3. **Dependency conflict resolution** — identified and fixed `pyarrow` version conflict with `mlflow`; identified and fixed Coinbase WebSocket URL mismatch
4. **Running the code** — Docker containers verified running; packages verified importable; WebSocket connection verified live

## What Was NOT Generated by AI

- The decision to use PR-AUC as the primary metric (course requirement)
- The threshold τ at 92nd percentile (based on EDA percentile plot)
- The choice of 60-second prediction horizon (course specification)
- The time-based train/val/test split rationale (no data leakage principle)
